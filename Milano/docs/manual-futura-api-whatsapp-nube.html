Manual de Futura Implementacion (API WhatsApp + Nube)

Manteniendo el sistema local original como respaldo.

1. Objetivo

Migrar gradualmente a una arquitectura cloud con API oficial de WhatsApp, sin perder la version local actual.

2. Principio clave: coexistencia

Local actual: se mantiene funcionando (modo principal o backup).

Nube nueva: se construye en paralelo y se activa por etapas.

3. Arquitectura recomendada (simple)

  • Servicio API: Node.js en nube (Render/Railway/VPS).
  • DB: Postgres o MySQL para usuarios, asignaciones y reportes.
  • Cola (opcional): para reintentos de envio.
  • Storage: logs y respaldos.
  • Panel: mismo frontend, cambiando base URL a API cloud.

4. Fases de implementacion

FaseAccionResultado
Fase 1Separar config local/cloud con variables env.Codigo preparado para dos entornos.
Fase 2Mover estado (asignaciones/procesados) a DB cloud.Reportes mas robustos.
Fase 3Integrar API oficial WhatsApp (Cloud API).Canal productivo escalable.
Fase 4Habilitar trafico gradual cloud.Migracion segura sin corte.

5. Mantener el local como respaldo

  • No borrar index.js local ni auth_info_baileys/.
  • Documentar comandos de arranque local de emergencia.
  • Tener export diario de logs y estado.
  • Definir procedimiento de rollback: cloud - local.

6. Checklist tecnico minimo para nube

  1. HTTPS habilitado.
  2. Manejo de secretos en variables seguras.
  3. Rate limit y logs de auditoria.
  4. Backups automaticos de DB.
  5. Monitoreo basico (errores, latencia, uso).

7. Roles sugeridos para la etapa cloud

  • admin: opera chats, asigna, gestiona usuarios.
  • operador: toma y responde chats.
  • viewer: solo lectura/reportes.

8. Plan de continuidad

Si falla cloud:

  1. Desviar operacion al servidor local.
  2. Levantar node index.js local.
  3. Continuar atencion por panel local.
