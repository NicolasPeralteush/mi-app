Manual de Desarrollo (Bot + Panel)

Guia simple para mantenimiento local del proyecto.

1. Estructura del proyecto

  • index.js: backend principal (WhatsApp, API interna, auth admin, WS).
  • panel/index.html: estructura del panel.
  • panel/styles.css: estilos del panel.
  • panel/app.js: logica del frontend.
  • prompt.txt: prompt del asistente.
  • registro_chats.txt: log de conversaciones.
  • auth_info_baileys/: sesion local de WhatsApp.

2. Preparacion inicial local

  1. Abrir terminal en carpeta del proyecto.
  2. Instalar dependencias: npm install
  3. Configurar .env (API key + admin login).
  4. Ejecutar: node index.js

3. Variables importantes en .env

GROQ_API_KEY=...
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=hash_sha256
PANEL_PORT=3000
ADMIN_TIME_ZONE=America/Argentina/Cordoba

4. Desarrollo del backend

Rutas clave

  • POST /admin/login, GET /admin/me
  • POST /admin/claim, /admin/release, /admin/processed
  • POST /send, /pause, /resume (protegidas)

Eventos WS clave

  • init, status, message, incoming
  • admin_assignment, admin_processed, admin_stats

5. Desarrollo del frontend (panel)

  • Modificar HTML en panel/index.html (estructura).
  • Modificar estilos en panel/styles.css.
  • Modificar logica en panel/app.js.
  • Probar flujo login - tomar chat - pausar - enviar - procesar.

6. Crear un admin nuevo

  1. Generar hash SHA256 de la clave.
  2. Actualizar .env con ADMIN_USERS_JSON o admin unico.
  3. Reiniciar servidor.

7. Usuarios no-admin (estado actual)

Actualmente no hay rol viewer. Solo existe acceso admin.

Si se requiere, implementar roles por usuario y chequeo por endpoint.

8. Checklist de pruebas antes de entregar

  • Login correcto e incorrecto.
  • Dos admins en paralelo (asignacion y bloqueo de acciones).
  • Pausa/reanudar y envio manual.
  • Marcado de procesado y cambios en reportes.
  • Reconectar WhatsApp y persistencia de estado.
